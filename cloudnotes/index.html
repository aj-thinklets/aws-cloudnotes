Amazon Simple Storage Service (S3)

Amazon S3 is object storage service that offers high scalability, data availability, security and performance. It offers (11 9’s) 99.999999999% durability.

What does 99.999999999% (11 9’s) mean?

If you store 10,000 objects on Amazon S3 on an average you may loose one of them every 10 million year or so.

S3 Terminologies

Namespace

Bucket name should be DNS compliant meaning,

  • Be unique across all of Amazon S3
  • Be between 3 and 63 characters long
  • Not contain uppercase characters
  • Start with a lowercase letter or number

S3 Storage classes

Amazon S3 offers a range of storage classes designed for different use cases.

Capacity

The total volume of data and number of objects you can store are unlimited. The individual S3 Object (document, video, photos etc.) can range in size from a a minimum of 0 bytes to a maximum of 5 terabytes. You can upload 5 gigabytes (GB) in single PUT. For objects larger than 100MB Amazon recommendes using the Multipart upload.

Consistency

  • Strong read-after-write consistency for PUTs and DELETEs of objects
  • Updates to a single key are atomic; meaning you will get either old or the new data, but never partial or corrupt data.

High availability

Amazon S3 achieves high availability by replicating data across multiple servers within AWS data centers. If a PUT request is successful, your data is safely stored.

S3 Versioning

Versioning is a means of keeping multiple variants of an object in the same bucket. It protects your data from accedential deletion.

  • Deleting Object: Instead of removing it permanently, Amazon S3 inserts a delete marker, which becomes the current object version. You can always restore the previous version.
  • If you overwrite an object, it results in a new object version in the bucket.
  • If you have an object expiration lifecycle policy in your non-versioned bucket and you want to maintain the same permanent delete behavior when you enable versioning, you must add a noncurrent expiration policy. The noncurrent expiration lifecycle policy will manage the deletes of the noncurrent object versions in the version-enabled bucket.
